# Storybook状態・検証の選択ガイド

ユーザーの判断、レイアウト、アクセシビリティ、通信結果、権限、入力値のいずれかが変わる状態は独立Storyにする。単なる色違いはthemeパラメータでまとめ、意味の異なる状態は分離する。

| 状態                  | 推奨する検証                        |
| --------------------- | ----------------------------------- |
| Default / Variant     | args、主要な表示、公開イベント      |
| Loading / Empty       | aria-busy、操作可否、レイアウト保持 |
| Error / Retry         | MSWの4xx/5xx、再試行、説明文        |
| Disabled / Permission | 操作不能、フォーカス、権限表示      |
| 長文 / RTL / Mobile   | 折返し、はみ出し、viewport、方向    |
| 非同期データ          | 成功・空・遅延・失敗・キャンセル    |

playは決定的なユーザー操作、公開イベント、フォーカス、表示契約を担う。スクリーンリーダー固有の読み上げ、実端末のタッチ、ブラウザ差は手動またはE2Eに分け、担当と実施条件を記録する。

Visual Regressionは再現性のある重要状態だけを対象にし、日時・ネットワーク・フォント・アニメーションを固定する。振る舞い・a11y・レスポンスの問題を画像更新だけで完了にしない。
