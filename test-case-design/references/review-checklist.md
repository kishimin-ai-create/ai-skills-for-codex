# Test Case Review Checklist

## 契約

- [ ] 入力、出力、エラー、状態遷移、外部副作用を明記した
- [ ] 仮定と仕様確認事項を分離した
- [ ] 正常系、異常系、境界値、空値、重複を必要な範囲で網羅した

## 構造

- [ ] 適切なテストレベルを選択した
- [ ] outlineでは実装ロジックを書かずit.todoだけを使った
- [ ] implementでは既存のdescribe構造とケース名を維持した
- [ ] 各ケースが1つの主要な振る舞いを検証する

## 独立性と副作用

- [ ] テスト間で状態を共有していない
- [ ] DB・外部HTTP・ファイルの後片付けが保証されている
- [ ] 異常系で状態不変条件を検証した
- [ ] モックの利用理由とテストレベルが一致している

## 実行と報告

- [ ] 実行コマンドと結果を記録した
- [ ] 失敗をテスト・実装・仕様／環境に分類した
- [ ] 未実装、スキップ、残存リスク、次の確認事項を報告した
