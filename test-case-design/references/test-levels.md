# Test Levels and Doubles

## Unit

- 純粋関数、変換、バリデーション、状態遷移を高速に検証する。
- 依存は最小限のfake・stubを許可するが、公開契約を隠す過剰なモックは避ける。

## Integration

- Repository、DB、トランザクション、内部HTTP境界を実際に接続して検証する。
- テスト用DBまたはコンテナを使い、fixtureとクリーンアップをテストごとに管理する。

## Contract

- APIのリクエスト、レスポンス、エラー、ステータス、スキーマ互換性を検証する。
- 内部実装のモックではなく、契約サーバーやテスト用の実HTTP境界を優先する。

## E2E

- 主要な利用者フロー、認証、画面遷移、重要な失敗経路を少数のシナリオで検証する。
- Unitで検証済みの細かな分岐をE2Eへ重複させない。

## 選択基準

仕様のリスク、外部副作用、実行時間、環境の再現性を比較し、最小のテストレベルで十分な保証を得る。実DB・実HTTPが必要な契約を、内部モックだけで代替しない。
