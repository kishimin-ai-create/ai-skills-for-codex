## テスト戦略

自動検査、コードレビュー、キーボード操作、表示確認、支援技術による手動確認を組み合わせる。自動ツールだけで適合性を判断しない。

変更のリスクに応じて検証範囲を広げる。

| 変更 | 最低限の検証 |
| --- | --- |
| テキスト・スタイルのみ | セマンティクス、コントラスト、ズーム、強制色 |
| フォーム | ラベル、説明、必須、エラー、キーボード、読み上げ |
| モーダル・メニュー・タブ | APGキー操作、フォーカス移動、終了後フォーカス、読み上げ |
| SPA遷移・非同期更新 | タイトル、フォーカス、読み込み・成功・失敗通知 |
| 主要ユーザーフロー | 自動検査、キーボード、スクリーンリーダー、ズーム、エラー経路 |

## 静的解析と自動検査

- プロジェクト既定のHTML、JSX、テンプレート向けアクセシビリティlintを実行する。
- axe-coreなど、プロジェクトで採用済みの検査をコンポーネントまたはE2Eテストへ組み込む。
- ページ初期表示だけでなく、モーダル表示、検証エラー、展開状態など重要な状態でも検査する。
- 例外ルールを無効化する場合は、理由、影響範囲、代替検証を記録する。
- 自動検査が見つけられない、操作順、名前の適切さ、代替テキストの意味、フォーカス移動も確認する。

## コンポーネントテスト

- ロールとアクセシブルネームで要素を取得する。
- ラベルと入力の関連、説明、エラーメッセージ、状態属性を検証する。
- 表示テキストだけでなく、展開、選択、無効、現在位置など利用者へ公開される状態を検証する。
- キーボードイベントを実際の操作順に近い方法で送る。
- DOMの内部構造やCSSクラスではなく、利用者が観測する振る舞いをアサートする。
- `data-testid`は、セマンティッククエリで表現できない場合だけ使用する。

## キーボードテスト

主要フローをマウスなしで実施する。

1. ページ先頭から`Tab`と`Shift+Tab`で移動する。
2. フォーカス順序とフォーカス表示を確認する。
3. `Enter`、`Space`、矢印キー、`Home`、`End`、`Escape`を該当パターンに従って確認する。
4. キーボードトラップがないことを確認する。
5. オーバーレイ表示中に背面へフォーカスが移動しないことを確認する。
6. オーバーレイ終了後に、起点または次の論理的な位置へフォーカスが戻ることを確認する。
7. 追加、削除、並べ替え、ページ遷移後のフォーカス位置を確認する。

ブラウザーのアドレスバーなどへ意図的に移動できることと、アプリ内部のキーボードトラップを区別する。

## 表示テスト

- テキストを200%まで拡大し、切り取り、重なり、操作不能がないことを確認する。
- 400%相当のズームまたは320 CSS px幅相当でリフローを確認する。
- 長い翻訳、ユーザー入力、エラーメッセージでもレイアウトを確認する。
- WCAGのテキスト間隔条件を適用し、内容の欠落を確認する。
- ライト・ダークテーマ、ホバー、フォーカス、選択、無効、エラーの各状態でコントラストを確認する。
- OSまたはブラウザーの強制色・高コントラスト設定で確認する。
- `prefers-reduced-motion`有効時の動作を確認する。
- ポートレート・ランドスケープとタッチターゲットを確認する。

## スクリーンリーダーテスト

対象ユーザーとサポート方針に基づき、代表的なブラウザーとスクリーンリーダーの組み合わせを選ぶ。一般的な候補はNVDAとChromeまたはFirefox、VoiceOverとSafariであるが、プロジェクトのサポート範囲を優先する。

次を確認する。

- ページタイトル、言語、ランドマーク、見出し階層
- コントロールの名前、ロール、値、状態、説明
- 読み上げ順序と操作順序
- 入力ラベル、必須、補足、エラー
- モーダル開始・終了時のフォーカスと背景の扱い
- 展開、選択、並べ替え、読み込み、成功、失敗の通知
- 表のキャプション、行・列見出し、セルの関係
- 画像の代替テキストと装飾画像の無視

読み上げ文言の完全一致へ過度に依存せず、必要な意味と操作可能性が伝わることを確認する。

## 回帰テスト

- 修正した問題を再現する最小の自動テストを追加する。
- 自動化できない手動確認は、再現手順と期待結果をチェックリストへ残す。
- 共有コンポーネントの修正では、代表的な利用箇所と全状態を確認する。
- UIライブラリー更新時は、アクセシビリティツリーとキーボード操作の差分を確認する。
- E2Eでは正常系だけでなく、入力エラー、通信失敗、空状態を含める。

## 記録

検証記録に以下を含める。

- 対象URL、画面、コンポーネント、状態
- 基準と適合レベル
- ブラウザー、OS、表示倍率
- 支援技術とバージョン
- 自動ツールとバージョン
- キーボード操作手順
- 期待結果と実際の結果
- 未検証範囲と残存リスク
