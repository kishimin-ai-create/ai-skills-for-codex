# アクセシビリティレビュー・チェックリスト

レビュー対象に該当する項目を確認する。該当しない項目は理由を記録する。

## 文書構造

- [ ] ページタイトルと主言語が適切である。
- [ ] 見出しが内容の階層を表している。
- [ ] ランドマークとセクショニング要素が適切である。
- [ ] リストと表の関係が機械可読である。
- [ ] DOM順序、読み上げ順序、視覚順序が論理的である。

## 操作要素

- [ ] ネイティブHTML要素を優先している。
- [ ] 各操作要素に目的を識別できるアクセシブルネームがある。
- [ ] 可視ラベルとアクセシブルネームが矛盾していない。
- [ ] ロール、状態、値が実際のUIと一致している。
- [ ] 無効、選択、展開、現在位置を色だけに依存せず伝えている。

## キーボードとフォーカス

- [ ] すべての機能をキーボードで完了できる。
- [ ] フォーカス順序が論理的である。
- [ ] フォーカス表示が常に認識でき、固定要素に隠れない。
- [ ] 正の`tabindex`と意図しないキーボードトラップがない。
- [ ] モーダルや複合ウィジェットがAPGのキー操作に従う。
- [ ] 表示開始、終了、削除、遷移後のフォーカス位置が適切である。

## フォーム

- [ ] 各入力に永続的な可視ラベルがある。
- [ ] 補足、必須、入力形式、単位、制約が入力前に分かる。
- [ ] 関連する入力群を`fieldset`と`legend`などで表現している。
- [ ] エラーが該当入力と関連付けられている。
- [ ] エラーが問題と修正方法を説明している。
- [ ] エラー後も入力値が保持され、修正箇所へ移動できる。

## 画像・音声・動画

- [ ] 意味のある画像に文脈に合う代替テキストがある。
- [ ] 装飾画像が支援技術から除外されている。
- [ ] 複雑な画像に同等の詳細情報がある。
- [ ] 音声・動画に必要な字幕、トランスクリプト、音声解説がある。
- [ ] 自動再生や時間制限を利用者が制御できる。

## 表示

- [ ] 通常テキスト、大きなテキスト、UI部品が必要なコントラストを満たす。
- [ ] 情報を色、位置、形、音だけで伝えていない。
- [ ] 200%文字拡大と400%相当のズームで情報・機能を失わない。
- [ ] 文字間隔の上書き、長文、翻訳文で内容を切り取らない。
- [ ] 強制色・高コントラスト設定でも状態とフォーカスを認識できる。
- [ ] 動きの軽減設定を尊重し、危険な点滅がない。
- [ ] タッチターゲットの大きさまたは間隔が適切である。

## 動的更新

- [ ] 読み込み、成功、失敗、件数変更が必要に応じて通知される。
- [ ] ライブリージョンの緊急度と更新頻度が適切である。
- [ ] 通知が重複せず、重要な読み上げを妨げない。
- [ ] SPA遷移後にタイトル、見出し、フォーカスが適切に更新される。

## テストと報告

- [ ] 静的解析と自動検査を重要なUI状態で実行した。
- [ ] 主要フローとエラー経路をキーボードだけで確認した。
- [ ] ズーム、リフロー、文字間隔、強制色、動きの軽減を確認した。
- [ ] リスクに応じたスクリーンリーダーテストを実施した。
- [ ] 自動化できる回帰テストを追加した。
- [ ] 未検証範囲、環境、残存リスクを記録した。
- [ ] 正式監査なしに完全適合を断定していない。
