# Legacy Boundary

## 目的

新しいコードとレガシーコードを分離し、レガシーコードの設計が新しいコードへ広がることを防ぐこと。

## 手順

### 1. 境界を設ける

特性化テストによって現在の振る舞いを記録した後、新しいコードとレガシーコードを分離する境界を設ける。

新しいコードがレガシーコードへ直接依存する代わりに、境界となるインターフェースを介してアクセスする。

### 2. 依存の方向を維持する

依存の方向は、新しいコードから境界を通してレガシーコードへ向ける。

レガシーコードから新しいコードへ依存させない。

既存の呼び出し元への変更を抑え、既存コードの修正を小さくする。

### 3. 型を変換する

必要に応じて、境界で新しいコードの型とレガシーコードの型を変換する。

新しいコードは、レガシーコード固有の型や構造へ直接依存しない。

### 4. Anti-Corruption Layer を適用する

必要に応じて、レガシーコードと新しいコードの間に Anti-Corruption Layer を設ける。

Anti-Corruption Layer は、レガシーコードの設計が新しいコードへ広がることを防ぐための境界として使用する。

新しいコードは、定義されたインターフェースを通じてレガシーコードへアクセスする。

境界の実装が、新しいコードで使用する型とレガシーコードで使用する型を変換する。

新しいコードは、レガシーコードの内部実装へ直接依存しない。
