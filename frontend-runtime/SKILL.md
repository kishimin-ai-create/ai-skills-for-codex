# Frontend Runtime

## 目的

API 通信、プロキシ、認証、時刻表示を正しく実装し、実行時の不整合を防ぐこと。

## 手順

### 1. API 通信を確認する

アプリに Next.js のルートプロキシがある場合、ブラウザ側の API 呼び出しでは通常、相対パス `/api/...` を使用する。

ブラウザの Network パネルにフロントエンドのオリジンが表示されていることだけを、不具合と判断してはならない。

API またはプロキシの不具合を調査する場合は、以下をすべて確認する。

1. バックエンドへの直接アクセスで得られる JSON
2. プロキシ経由で得られる JSON
3. 実際にレンダリングされた UI の状態

`200` ステータスだけで正常と判断してはならない。

### 2. プロキシを確認する

サーバー側のプロキシコードでは、バックエンドのオリジンをランタイム設定から解決する。

バックエンドのオリジンがフロントエンドのリクエストオリジンと同一の場合は拒否する。

ランタイムがレスポンスボディを再送信する場合、`content-length` や `content-encoding` などのレスポンスフレーミングヘッダーを無条件に保持してはならない。

### 3. 認証を確認する

ログイン時にトークンを保存する場合は、ユーザーから見えるアプリケーションシェルにログアウト操作を用意する。

ログアウト時は、ログイン時に使用した同じストレージからトークンを削除し、ナビゲーションを更新する。

### 4. 時刻表示を確認する

API またはデータベースのタイムスタンプが UTC であり、UI でもそれに一致する表示が期待される場合は、`Intl.DateTimeFormat` で `timeZone` を明示的に指定する。

レンダリングされた時刻の値をテストで検証する。
