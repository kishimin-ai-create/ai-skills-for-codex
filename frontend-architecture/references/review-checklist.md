# Frontend Architecture Review Checklist

## 要件

- [ ] パブリック／プライベート、認証、SEO、初期表示要件を明記した
- [ ] 更新頻度、データ鮮度、トラフィック、可用性、コストを確認した
- [ ] 運用体制、リージョン、デプロイ、ロールバック制約を確認した

## 構成

- [ ] ルートごとのCSR／SSR／SSG／ISRの理由がある
- [ ] ホスティングとレンダリングの組合せを説明できる
- [ ] BFFの有無、転送先、相対パス、ループ防止を確認した
- [ ] SPAフォールバックと深いURLの直接アクセスを検証した

## セキュリティと運用

- [ ] 秘密情報と内部オリジンをブラウザへ露出していない
- [ ] HTTPS、CORS、セキュリティヘッダー、認証境界を確認した
- [ ] キャッシュ、レスポンスヘッダー、タイムアウト、ログを確認した
- [ ] 監視、障害時の挙動、ロールバックを定義した

## 記録

- [ ] 採用案、代替案、前提、残存リスクをADRまたは設計記録に残した
- [ ] 機能追加、性能改善、無関係なバグ修正を差分へ混在させていない
