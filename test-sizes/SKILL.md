# Test Sizes

## 目的

テストをリソース使用状況に応じて分類し、テストサイズをファイル名から明確に識別できるようにすること。

## ファイル命名規則

すべてのテストファイルは、ファイル名に**テストサイズを必ず含めること**。

| サイズ | パターン          | 例                        |
| ------ | ----------------- | ------------------------- |
| Small  | `*.small.test.*`  | `createApp.small.test.*`  |
| Medium | `*.medium.test.*` | `createApp.medium.test.*` |
| Large  | `*.large.test.*`  | `checkout.large.test.*`   |

- サイズを含まないテストファイルを作成してはならない。
- テストサイズは、以下の分類表に基づき、実際のリソース使用状況と一致させること。

## テストサイズの分類

| 特性                         | Small | Medium         | Large   |
| ---------------------------- | ----- | -------------- | ------- |
| ネットワークアクセス         | 不可  | localhost のみ | 可      |
| データベース                 | 不可  | 可             | 可      |
| ファイルシステムへのアクセス | 不可  | 可             | 可      |
| 外部システムの使用           | 不可  | 非推奨         | 可      |
| 複数スレッド                 | 不可  | 可             | 可      |
| sleep 処理                   | 不可  | 可             | 可      |
| システムプロパティ           | 不可  | 可             | 可      |
| 制限時間（秒）               | 60    | 300            | 900以上 |

## ルール

- Medium テストより Small テストを多くし、Large テストより Medium テストを多くする。
