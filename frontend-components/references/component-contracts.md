# Frontend Component

## 目的

コンポーネントの責務を明確に分離し、ネイティブ HTML の標準仕様と互換性のある再利用可能なコンポーネントを設計すること。

## 手順

### 1. コンポーネントの責務を決定する

コンポーネントを作成または変更する前に、その責務を確認し、適切な場所に配置する。

フォルダ名（`ui/`、`features/`、`pages/`）から責務が明確に分かるようにする。

ページレベルのロジックを `ui/` に配置してはならない。

### 2. UI コンポーネントの依存関係を確認する

UI レベルのコンポーネントから、以下を直接 import してはならない。

- グローバルステートのフック
- ルーティングのフック
- データフェッチロジック

これらの依存関係は `features/` または専用のラッパーに移動する。

### 3. 共有 UI コンポーネントの props を設計する

共有 UI コンポーネントは、ネイティブ HTML の props とイベントハンドラーを拡張し、そのまま渡せるようにする。

以下のような標準 props を独自のシグネチャで再定義してはならない。

- `onClick`
- `disabled`
- `type`
- `aria-*`
