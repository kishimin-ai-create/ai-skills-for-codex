# Legacy Change Techniques

## 目的

既存コードへの変更を最小限に抑えながら、新しい振る舞いを追加すること。

## 適用条件

変更による影響を最小限に抑えたい場合や、既存コードへの直接的な変更を避けたい場合は、以下の技法を検討する。

## Sprout Method

既存のメソッドへ新しい処理を直接追加する代わりに、新しい振る舞いを新しいメソッドとして作成する。

新しいメソッドをテストし、既存コードから呼び出す。

## Sprout Class

新しい振る舞いを既存のクラスへ直接追加する代わりに、新しいクラスとして作成する。

新しいクラスをテストし、既存コードから使用する。

## Wrap Method

既存のメソッドをラップし、既存の振る舞いの前後に新しい振る舞いを追加する。

## Wrap Class

既存のクラスをラップし、既存クラスへの変更を抑えながら新しい振る舞いを追加する。

## 差分プログラミング

既存コードを直接大きく変更する代わりに、既存コードとの差分として新しい振る舞いを追加する方法を検討する。

既存の振る舞いを利用しながら、変更部分を分離する。
