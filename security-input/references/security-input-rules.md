# Security: Input Rules

## Encoding and Output Escaping (V1)

- 入力を処理前に一度だけデコードまたはアンエスケープし、検証・サニタイズ後に再度行わない。
- インタープリターへ渡す直前を最終段階として出力エンコードするか、インタープリターに任せる。
- HTML要素、HTML属性、CSS、JavaScript、URLパラメーターごとにコンテキスト対応エンコードを使う。

## Injection Prevention (V1.2)

- SQL、HQL、NoSQL、Cypher（ストアドプロシージャを含む）はすべてパラメーター化クエリまたはORMを使う。
- OSコマンドはパラメーター化APIまたはコンテキスト対応エンコードを使い、ユーザー入力をシェルコマンドへ連結しない。
- JavaScriptとJSONの動的出力をエンコードし、JS/JSONインジェクションを防ぐ。
- URLのクエリとパスの値をURLエンコードまたはbase64urlで処理し、`javascript:`や`data:`を含む危険なプロトコルを許可しない。
- LDAP、XPath、LaTeX、CSV・数式インジェクションには適切なエスケープまたは許可リストを使う。

## Sanitization (V1.3)

- WYSIWYG/HTML入力はDOMPurifyなどの実績あるライブラリでサニタイズし、自作HTMLパーサーを作らない。
- `eval()`と動的コード実行を避ける。不可避なら実行前に厳格なサニタイズを行う。
- 信頼できないURLをプロトコル、ドメイン、パス、ポートの許可リストで検証し、SSRFを防ぐ。
- ユーザー提供のテンプレート、Markdown、CSS、BBCodeをサニタイズまたは無効化する。
- ユーザー提供SVGからスクリプトと`<foreignObject>`要素を除去する。
- バックトラッキングが発生しやすい正規表現を避け、正規表現エンジンへ渡す前に入力を制限してReDoSを防ぐ。

## Safe Deserialization (V1.5)

- すべてのXMLパーサーで外部エンティティ解決を無効化し、XXEを防ぐ。
- 信頼できないデータは許可リストの型だけへデシリアライズし、本質的に安全でない方式を使わない。
- 同一データ型には一貫したパーサーを使用し、JSON/URIの相互運用攻撃を防ぐ。

## Input Validation (V2)

- すべての入力を信頼されたサーバー側層で検証する。クライアント検証はUX目的だけとする。
- 型、形式、範囲、長さを使用前に正の許可リストで検証する。
- 郵便番号と地域など、組み合わせたフィールドの文脈的一貫性を検証する。
- ユーザー単位と全体の業務上限を強制し、悪用を防ぐ。

## Business Logic (V2.3)

- 手順を段階的に強制し、ユーザーがステップを飛ばせないようにする。
- 複数段階の操作はデータベーストランザクションで原子性を保つ。
- 有限資源（席、枠、在庫）の二重予約を防ぐためロックを使う。
- 高価値操作では複数ユーザーの承認を要求する。

## Anti-automation (V2.4)

- クレデンシャルスタッフィング、データ流出、クォータ濫用を防ぐため、レート制限と自動化対策を適用する。
- 機密取引の送信には、現実的な人間の操作時間を要求する。

## File Handling (V5)

- 許可された拡張子とContent-Typeだけを受け付け、拡張子だけでなくマジックバイトで内容を検証する。
- アップロードサイズとユーザーごとのファイル数クォータを制限する。
- 展開前にzipやgzなどの圧縮アーカイブを検査し、展開サイズとファイル数で爆弾攻撃を防ぐ。
- HTTPでアクセスされたアップロードファイルをサーバー側コードとして実行しない。
- ファイルパスはユーザー指定名ではなく、信頼された内部データから組み立て、パストラバーサル、LFI、SSRFを防ぐ。
- ダウンロードには安全なファイル名を指定した`Content-Disposition`を付与する。
- 信頼できないソースのファイルをウイルス対策でスキャンしてから提供する。
- 許可上限を超えるピクセル寸法の画像を拒否し、ピクセルフラッド攻撃を防ぐ。

## Review Evidence

- 各インタープリター向けのパラメーター化・エンコード、サーバー側許可リスト、上限設定を確認する。
- インジェクション、SSRF、XXE、ReDoS、業務フロー逸脱、レート制限の拒否テストを確認する。
- ファイルのマジックバイト、展開制限、実行防止、ウイルススキャン、画像寸法制限を確認する。
