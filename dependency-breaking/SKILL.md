# Dependency Breaking

## 目的

テストを妨げる依存関係を特定し、既存の振る舞いを変えずに対象コードをテスト可能な状態へ分離すること。

## 手順

### 1. テストを妨げる依存関係を特定する

対象コードをテストハーネスの中で動かせるか確認する。

動かせない場合は、原因となる依存関係を調査する。

確認対象には次のようなものが含まれる。

- 扱いにくいパラメータ
- 隠れた依存関係
- 複雑なオブジェクト生成
- グローバルな依存関係
- `include` の依存関係
- 多段階に渡されるパラメータ
- 別名を持つパラメータ
- 隠れたメソッド
- 言語機能によるテスト困難性
- 検出できない副作用
- ライブラリへの強い依存
- API 呼び出し

### 2. 検出可能か確認する

テストから対象コードの結果を検出できるか確認する。

戻り値だけでなく、必要に応じて次のような結果も確認する。

- 状態の変化
- 出力
- 副作用
- 外部システムへの呼び出し

結果を検出できない場合は、検出できる場所を作る。

### 3. 分離可能か確認する

対象コードを、他のコードや外部システムから分離して実行できるか確認する。

分離できない場合は、対象コードを単独で実行できる場所を探す。

### 4. 接合部を見つける

既存コードの振る舞いを変更せず、依存関係を置き換えられる場所を探す。

既存の接合部を利用できる場合は、その接合部を使用する。

既存の接合部がない場合は、テストに必要な範囲で接合部を作る。

### 5. 依存関係を排除する

テストを書くことや変更することを妨げる依存関係を排除する。

依存関係の排除は、コードをテストハーネスに入れ、フィードバックを得ながら変更できるようにするために行う。

すべての依存関係を無条件に排除してはならない。

変更に必要な箇所についてのみ行う。

### 6. 振る舞いが変わっていないことを確認する

接合部の作成や依存関係の排除を行った後、既存の振る舞いが変わっていないことを確認する。

既存テストまたは特性化テストを実行し、成功することを確認する。

### 7. テストできない箇所を記録する

依存関係を排除できず、テストを書けない箇所が残る場合は、その箇所を記録する。

テストできない原因となっている依存関係や構造を、後の改善対象として残す。
