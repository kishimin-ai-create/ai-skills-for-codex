# 原因・修正選択ガイド

| 観測                       | まず確認                      | 修正の方向                         |
| -------------------------- | ----------------------------- | ---------------------------------- |
| 期待statusと実statusが違う | ドメインエラー分類と境界変換  | エラーを分類して契約へ変換         |
| 200だが表示が壊れる        | Content-Type、parse、schema   | body検証後に状態を決定             |
| 古い結果が新結果を上書き   | 要求順序、Abort、cache key    | 世代管理・キャンセル・キー修正     |
| テストだけ失敗             | 再現環境、fixture、時刻、順序 | 決定性を回復し、テストを弱めない   |
| 本番だけ失敗               | 設定、依存、権限、データ差    | 差分を最小再現し、設定を安全に修正 |
| UI回帰                     | role、label、focus、viewport  | 利用者契約と状態を回帰テスト       |

原因が確定しない場合は、仮説、追加観測、停止条件を記録して断定しない。大規模なリファクタリングや依存更新を、局所的な不具合修正に便乗させない。
