# Frontend Security Review Checklist

## 資産と境界

- [ ] 保護対象のデータ、秘密、権限、ログを分類した
- [ ] ブラウザ、API、BFF、IdP、CDN、開発環境の信頼境界を確認した
- [ ] XSS、CSRF、IDOR、Open Redirect、依存関係侵害を検討した

## 認証・認可

- [ ] 認証と認可を分離し、認可をサーバーで再検証した
- [ ] Cookieまたはトークンの期限、失効、ローテーションを定義した
- [ ] state、nonce、PKCE、return URLを検証した
- [ ] 期限切れ、ログアウト、再認証、複数タブを確認した

## 入力・ブラウザ・依存

- [ ] 入力をサーバー側で型・形式・権限まで検証した
- [ ] 出力コンテキストごとにエスケープした
- [ ] CSRF、CORS、CSP、HSTS、主要Security Headerを確認した
- [ ] lockfile、脆弱性スキャン、依存更新方針を確認した
- [ ] エラー、ログ、URL、分析イベントに秘密情報がない

## 検証と対応

- [ ] 正常・異常・権限境界・失効・再試行をテストした
- [ ] 検出、監査、失効、ロールバック、インシデント対応を記録した
- [ ] 非対応範囲、残存リスク、追加確認事項を報告した
