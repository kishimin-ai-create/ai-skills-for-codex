# Characterization Testing

## 目的

変更対象の現在の実際の振る舞いを記録し、変更による差分を検出できるようにすること。

## 手順

### 1. テスト対象を決める

変更対象を保護するために、どのコードを特性化テストの対象にするか決める。

変更するコードそのものだけでなく、変更による影響を検出できる場所を確認する。

対象のコードをテストハーネスの中で呼び出せるか確認する。

テストハーネスに入れられない場合は、その原因となる依存関係を確認する。

### 2. 現在の振る舞いを観測する

特性化テストでは、コードの「あるべき振る舞い」ではなく、「現在の実際の振る舞い」を記録する。

現在の振る舞いが正しいかどうかは、この段階では問わない。

バグと思われる振る舞いが存在する場合でも、現在そのように動いているのであれば、その振る舞いを記録する。

現在の振る舞いを記録することと、その振る舞いを修正することを区別する。

### 3. 特性化テストを書く

以下の手順で特性化テストを作成する。

1. テストハーネスの中で対象のコードを呼び出す。
2. 失敗すると分かっている表明を書く。
3. テストを実行する。
4. 失敗した結果から実際の振る舞いを確認する。
5. コードが実現する振る舞いを期待するようにテストを変更する。
6. この手順を繰り返す。

### 4. 入力を選定する

変更対象に応じて、現在の振る舞いを確認するための入力を用意する。

対象には次のようなものが含まれる。

- 通常の入力
- 複数の要素を含む入力
- 境界値
- `0`
- `1`
- 最大値
- `null`
- 無効な入力
- エラーになる入力
- 特殊な条件
- 例外的な条件

### 5. 期待値を確定する

期待値には、実際にコードを実行して得られた結果を使用する。

推測した仕様や理想的な振る舞いを期待値にしてはならない。

予期しない振る舞いを発見した場合は、その内容を明確にする。

バグの可能性がある場合は疑わしいものとして扱い、その部分を修正した場合の影響を調査する。

### 6. 現在の振る舞いを文書化する

特性化テストによって、コードの現在の実際の振る舞いを文書化する。

作成したテストが、変更前のコードに対して成功することを確認する。
