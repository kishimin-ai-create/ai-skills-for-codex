# 記事レビュー・チェックリスト

記事の種別と公開範囲に該当する項目を確認し、未確認の項目は理由を残す。

## 目的と読者

- [ ] 対象読者、前提知識、想定環境が明示されている。
- [ ] 読者の課題と記事の結論が一致している。
- [ ] 扱うこと・扱わないことが分かる。
- [ ] 読了後に期待する行動または判断が分かる。
- [ ] 記事種別と構成が目的に合っている。

## 構成と文章

- [ ] タイトルが内容を正確に表している。
- [ ] 冒頭で結論、対象、前提が分かる。
- [ ] 見出し階層が論理的で、1節1論点になっている。
- [ ] 手順に前提、操作、期待結果、失敗時の確認がある。
- [ ] 事実、推測、意見、未検証事項が区別されている。
- [ ] 同じ用語、表記、単位、日付形式を一貫して使っている。
- [ ] 結論、本文、まとめで約束が矛盾していない。

## 根拠と再現性

- [ ] 変化し得る情報に版、環境、確認日がある。
- [ ] 重要な主張に一次資料または検証結果がある。
- [ ] リンクが実在し、目的のページへ直接到達する。
- [ ] コード、コマンド、入力、期待結果が追試可能である。
- [ ] 実行していない内容を実行済みと書いていない。
- [ ] 失敗条件、制約、代替策、ロールバックが必要に応じて示されている。
- [ ] 推奨や比較に評価軸、トレードオフ、再考条件がある。

## コードとデータ

- [ ] コードフェンスに言語識別子がある。
- [ ] コード、識別子、パス、コマンド、URL、プレースホルダーが正しい。
- [ ] 省略部分と実行可能部分が区別されている。
- [ ] 依存バージョンと実行ディレクトリが分かる。
- [ ] 秘密情報、個人情報、内部URL、実在認証情報がない。
- [ ] 破壊的操作、権限変更、外部送信、課金操作へ警告がある。
- [ ] 数値、単位、通貨、タイムゾーン、測定条件が明示されている。

## 図・表・アクセシビリティ

- [ ] 図や画像だけに重要情報を置いていない。
- [ ] 画像の代替テキストまたは本文の要約がある。
- [ ] 表の見出し、単位、比較軸が明確である。
- [ ] リンクテキストが目的を表している。
- [ ] 色だけで状態、エラー、重要度を示していない。
- [ ] 長いコードや表が読み上げ・ズーム時にも追跡できる。

## 媒体と公開

- [ ] 対象媒体のMarkdown、HTML、文字数、リンク形式に従っている。
- [ ] 公開範囲に対して機密情報と内部参照を確認している。
- [ ] リリースノートなら利用者への影響、移行、既知の問題がある。
- [ ] 開発ログなら判断、失敗、未完了事項、次の作業がある。
- [ ] 更新日が内容の変更を反映している。
- [ ] 公開後に古くなる情報と更新担当が分かる。

## 最終報告

- [ ] 選択した記事種別と理由を記録した。
- [ ] 参照資料と確認日を記録した。
- [ ] 実施した検証と結果を記録した。
- [ ] 未検証範囲、既知の制約、残存リスクを記録した。
- [ ] 形式的な検証をしていないのに、保証や完全性を断定していない。
