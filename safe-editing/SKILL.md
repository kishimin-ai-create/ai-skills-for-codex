# Safe Editing

## 目的

レガシーコードの変更による影響を最小限に抑え、安全に変更を進めること。

## 手順

### 1. 超集中編集を行う

変更対象へ集中し、無関係な変更を混ぜない。

### 2. 単一目的の編集を行う

1回の編集で複数の目的を混在させない。

変更は1つの目的に限定する。

### 3. シグネチャを維持する

変更中は既存のシグネチャを維持し、影響範囲を抑える。

必要以上に公開 API や呼び出し元へ影響を与えない。

### 4. コンパイラを活用する

コンパイラによるフィードバックを利用し、変更の影響を確認する。

### 5. 変更を確認する

変更後は、既存の振る舞いが壊れていないことを確認する。
