# Write Article

このスキルは、リポジトリ内で共有される

## Use This Skill When

- ユーザーが `/write-article` を依頼した場合
- リポジトリの変更内容をもとにした技術記事や開発ログ（Devlog）の作成を求められた場合
- 出力先を `blog/` 配下とする場合

## Workflow

3. Sub-agentが利用できない場合は、同じ「観測可能な事実のみを扱う」ルールに従って直接作業を行う。
4. まず、ユーザーの依頼内容から記事の対象範囲を特定する。
5. 対象範囲が十分でない場合は、関連する最新のリポジトリ変更、関連ドキュメント、および現在のWorking Treeの変更内容から推測する。
6. 完成したMarkdownファイルを `blog/` 配下へ保存する。
7. チャットでは、生成したファイルパスと記事の対象範囲を報告する。

## Output Expectations

- 日本語の技術記事として執筆する
- 背景、変更内容、実装の詳細、および主な学びを分かりやすく説明する
- 不要な前置きや冗長な説明は最小限にする
- 根拠のない主張は記載しない
