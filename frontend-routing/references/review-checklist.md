# Frontend Routing Review Checklist

## URLと遷移

- [ ] 公開、認証、権限、API、404、エラーのルートを一覧化した
- [ ] 動的セグメント、クエリ、ロケール、末尾スラッシュの規則を定義した
- [ ] 直接URL、再読み込み、戻る・進む、共有URLを検証した

## セキュリティ

- [ ] 認証・認可をサーバー境界で検証した
- [ ] return URLを相対パスまたはallowlistに制限した
- [ ] URL、ログ、エラー画面へ秘密情報を出していない
- [ ] 401、403、セッション期限切れを区別した

## データと表示

- [ ] loading、empty、404、forbidden、errorを区別した
- [ ] Abort、重複取得、古いレスポンス、キャッシュを確認した
- [ ] 選択したSSR／SSG／CSR／再検証の理由を記録した

## SEO・運用・検証

- [ ] title、canonical、robots、sitemap、OGPを確認した
- [ ] ルート単位のテストと主要なE2E遷移を実行した
- [ ] 変更範囲、ロールバック、残存リスク、非対応範囲を記録した
