# サマリー品質チェックリスト

- [ ] 冒頭に完了状態と結論がある
- [ ] 変更ファイルと目的が漏れなく列挙されている
- [ ] 実行した検証コマンドと結果がある
- [ ] 未実行の検証を成功のように書いていない
- [ ] FACT、ASSUMPTION、INFERENCEを区別している
- [ ] 失敗・ブロック・残存リスクを隠していない
- [ ] テストが表現するWhatと、コード変更のHowを混同していない
- [ ] コミットやpushを実行したと誤認させる表現がない
- [ ] 機密情報、トークン、個人情報、不要なログを含めていない
- [ ] 次の担当者が追加質問なしで再開できる
